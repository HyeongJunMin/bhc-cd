# 실행 백로그 상세 가이드 (Codex 세션 인수인계용)

## 1. 문서 목적
- 이 문서는 "다른 Codex 세션"이 들어와도 바로 실행 가능한 작업 단위를 제공한다.
- 단순 TODO가 아니라, 각 작업의 목적/입출력/완료조건/검증 명령을 함께 기록한다.
- 더 작은 실행 단위가 필요하면 `docs/Execution-Backlog-Micro.md`를 우선 사용한다.
- 기준 문서는 아래와 같다.
  - 제품 요구사항: `docs/GDD.md`, `docs/GDD_kr.md`
  - 물리 명세: `docs/Physics-Spec.md`, `docs/Physics-Spec_kr.md`
  - 입력 계약: `docs/Input-Schema.md`, `docs/Input-Schema_kr.md`, `schemas/shot-input-v1.json`
  - 기술 스택: `docs/Tech-Stack.md`

## 2. 세션 시작 체크리스트 (반드시 먼저 수행)
1. `AGENTS.md`를 읽고 Spec Sync Rules를 확인한다.
2. 이번 작업이 스펙 변경인지 구현 변경인지 분류한다.
3. 스펙 변경이면 연관 문서 동시 수정 계획을 먼저 세운다.
4. 구현 변경이면 아래 백로그에서 가장 작은 단위를 선택한다.
5. 작업 종료 전에 검증 명령을 실행하고 결과를 기록한다.

## 3. 공통 완료 기준 (Global DoD)
- 코드가 로컬에서 빌드/테스트 가능한 상태다.
- 관련 문서가 최신 상태다.
- 스펙 파일을 바꿨다면 `python3 scripts/ci/spec_guard.py <changed_files...>` 통과.
- 에러/예외 플로우를 최소 1개 이상 검증했다.
- 커밋/PR 설명에 "무엇을 왜 바꿨는지"와 "어떻게 검증했는지"가 포함된다.

## 4. 우선순위 규칙
- P0: 게임 규칙 위반/입장 잠금 위반/턴 로직 오류/점수 종료 오류
- P1: 인증/로비/방장 권한/입력 검증
- P2: 성능 최적화/UX 개선/운영 편의
